## ✨ Overview

AI Insight Lab is a production-grade web application built for Stella College that enables educators to upload student assignments and receive instant AI-powered analysis. The platform detects AI-generated content, identifies potential plagiarism from internet sources, and — for Q&A-format submissions — evaluates the correctness of student answers.

The app leverages a dual-provider AI strategy (Google Gemini 2.0 Flash + Groq LLaMA 3.1) with automatic failover, SHA-256 content-based caching via Supabase, and parallel multi-file processing to deliver fast, reliable, and cost-efficient analysis at scale.

## 🚀 Key Features

| Feature | Description |
|---|---|
| 🤖 AI Content Detection | Determines what percentage of a submission was generated by AI (ChatGPT, Gemini, etc.) |
| 🌐 Internet Plagiarism Check | Identifies content that closely matches material found on the web |
| 👤 Human Authorship Score | Calculates the proportion of genuinely human-written content |
| 📊 Confidence Rating | Provides an overall confidence level for the analysis |
| ❓ Q&A Evaluation | For question-and-answer formatted docs, evaluates each answer as Correct / Partially Correct / Incorrect |
| 📁 Multi-File Upload | Upload and analyze up to 10 files simultaneously with drag-and-drop |
| ⚡ Parallel Processing | All files are analyzed concurrently via individual edge function invocations |
| 💾 SHA-256 Caching | Previously analyzed files are instantly served from cache (no re-analysis cost) |
| 📄 PDF Report Generation | Export a professional, multi-page PDF report with cover page, aggregate summary, and per-file details |
| 🔐 Google OAuth | Secure sign-in via Google through Supabase Auth |
| 🎯 Dual AI Providers | Gemini 2.0 Flash as primary, Groq LLaMA 3.1 as fallback — automatic failover |
| 🎨 Glassmorphism UI | Modern dark-mode interface with animated glass-card components and smooth Framer Motion transitions |

### Data Flow

1. User uploads files → FileUpload component validates type/size → adds to `entries[]` state
2. User clicks "Analyze" → For each file in parallel:
   - `fileParser.ts` extracts text (PDF via `pdfjs-dist`, DOCX via `mammoth`, TXT natively)
   - SHA-256 hash is computed using Web Crypto API
   - Supabase edge function is invoked with `{ fileHash, fileName, text }`
3. Edge function → Checks `pdf_analysis_cache` by `file_hash`:
   - Cache HIT → Returns cached result instantly (marked `cached: true`)
   - Cache MISS → Calls Gemini 2.0 Flash (primary) or Groq LLaMA 3.1 (fallback)
   - Stores result in cache for future lookups
4. Results rendered → CircleProgress animations, tabbed multi-file view, Q&A evaluation
5. PDF export → `jsPDF` generates a professional multi-page report with cover, aggregate summary, and per-file details

## ⚙️ Getting Started

### Prerequisites

- Node.js ≥ 18
- npm, bun, or pnpm
- A Supabase project (free tier works)
- API keys for Gemini and/or Groq

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/your-org/ai-insight-lab.git
cd ai-insight-lab

# 2. Install dependencies
npm install
# or
bun install

# 3. Set up environment variables
cp .env.example .env
# Edit .env with your credentials (see section below)

# 4. Start the development server
npm run dev
```

The app will be available at `http://localhost:8080`.

### Supabase Setup

1. Create a new Supabase project at [supabase.com](https://supabase.com)
2. Create the cache table:

```sql
CREATE TABLE pdf_analysis_cache (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  file_hash TEXT UNIQUE NOT NULL,
  file_name TEXT NOT NULL DEFAULT 'unknown',
  full_report JSONB NOT NULL,
  ai_percentage NUMERIC,
  human_percentage NUMERIC,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_file_hash ON pdf_analysis_cache(file_hash);
```

3. Enable Google OAuth in Supabase Auth → Providers → Google
4. Deploy the edge function:

```bash
supabase functions deploy analyze-assignment
```

5. Set edge function secrets:

```bash
supabase secrets set GEMINI_API_KEY=your_gemini_key
supabase secrets set GROQ_API_KEY=your_groq_key
```

## 🔌 API & Edge Functions

### `analyze-assignment` Edge Function

Endpoint: `POST /functions/v1/analyze-assignment`

Request Body:
```json
{
  "fileHash": "a1b2c3d4e5f6...",
  "fileName": "assignment.pdf",
  "text": "Extracted text content from the document..."
}
```

Response (Success):
```json
{
  "aiPercentage": 35,
  "humanPercentage": 50,
  "internetPercentage": 15,
  "confidence": 82,
  "flaggedSections": ["This section appears to follow typical AI patterns..."],
  "recommendations": ["Consider adding more personal analysis..."],
  "summary": "The submission shows moderate AI assistance...",
  "isQnA": false,
  "questionsAnalysis": [],
  "cached": false
}
```

Response (Cached):
Same structure with `"cached": true`.

Response (Fallback/Error):
```json
{
  "error": "Both AI providers failed.",
  "fallback": true
}
```

### AI Provider Strategy

```
Request ──▶ Cache Check ──▶ HIT? ──▶ Return cached result
                │
                ▼ MISS
          Gemini 2.0 Flash
                │
           Success? ──▶ Save to cache ──▶ Return result
                │
                ▼ FAIL
          Groq LLaMA 3.1
                │
           Success? ──▶ Save to cache ──▶ Return result
                │
                ▼ FAIL
          Return fallback error
```
